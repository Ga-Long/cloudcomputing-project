Service-Storage

Amazon Simple Storage Service(S3)

S3

  • 데이터 저장 및 검색
  • 속도, 내구성, 가용성이 뛰어난 객체 액세스
  • 버킷에 저장할 수 있는 객체 수 무제한
  • 웹 어디서나 액세스 : url을 통한 공유
  • 정적 웹 사이트 호스팅
  • storage마다 가격이 다름
  • Intelligent-Tiering : Data Access 패턴이 변경될때, 성능에 대한 영향 없이 스토리지 비용을 자동으로 최적화
  • 객체별 월별 모니터링 및 자동화를 통해 자주 조회(Frequent)하거나, 자주 조회하지 않는(Infrequent) 객체들의 티어를 자동으로 조정 해준다 → 자동 비용 최적화

S3 Glacier

  • 저렴한 데이터 아카이빙 및 장기 백업
  • S3 glacier에 S3 콘텐츠의 수명 주기 아카이빙을 구성할 수 있음

S3

  • 사용한 만큼 지불
    • 월별 GB
    • 다른 리전으로 송신
    • PUT, COPY, POST, LIST 및 GET 요청
  • 비용 지불하지 않음
    • S3로 수신하는 경우
    • S3에서 같은 리전의 CloudFront 또는 EC2로 송신하는 경우

Amazon Elastic Block Store(EBS)

1GB 파일에서 글자 하나를 변경하려고 한다면 ❓

  • 블록 스토리지
    • 해당 글자가 포함된 블록 하나를 변경
  • 객체 스토리지
    • 전체 파일을 업데이트

EBS

  • 인스턴스용 ( 서버형 스토리지) 영구 블록 스토리지
  • 자동 복제를 통해 보호
  • 프로비저닝한 만큼만 요금 지불
  • 스냅샷 → S3에 저장, 자동 백업
  • 용도
    • EC2 인스턴스 부팅 볼륨(OS 정보를 가지고 있어 바로 시작가능) 및 스토리지
    • 파일 시스템의 데이터 스토리지
    • …볼륨을 분리하고 다른 EC2 인스턴스에 다시 연결을 해도 다시 처음부터 os를 설치할 필요없이 바로 똑같은 환경이 됨.
  • EC2에 attach해서 이용
    • 단일 AZ에서 여러 서버에 걸처 복제
    • 볼륨 스냅샷 → S3에 저장되어 복수 AZ에 자동 복제

Service-DB

AWS Database 종류

DIY(Do It Yourself) vs AWS DB

  • 비관리형 : 사용자가 확장, 내결함성 및 가용성을 관리 (Amazon EC2의 DB)
  • 관리형 : 확장, 내결함성 및 가용성이 서비스에 내장되어 있음 (AWS DBS)

Amazon RDS

데이터 베이스는 파란색!

적합하지 않은 사례

  • 대규모 읽기/쓰기 (예: 초당150k)
  • 큰 데이터 크기 또는 처리량
  • NoSQL 데이터베이스가 처리할 수 있는 간단한 GET/PUT 요청 및 쿼리
  • 원하는 대로 바꾼 RDBMS

시간단위 결제

  • 실행할 때 리소스 요금 발생
  • 주의 : 7일 후 자동 재시작

프로비저닝 된 스토리지( 인스턴스와 연결한 경우)

  • 무료 : 활성 데이터베이스에 사용되는 데이터베이스 스토리지의 최대 100%에 해당하는 백업 스토리지
  • 청구(GB/ 월) : 종료된 DB 인스턴스의 백업 스토리지

Amazon Aurora

: 클라우드용으로 빌드된 MySQL 및 PostgreSQL 호환 관계형 데이터베이스

Amazon DynamoDB

: 빠르고 유연한 NoSQL 데이터베이스 서비스

Amazon Redshift

: 가장 빠르고 가장 쉽고 가장 널리 사용되는 클라우드 데이터 웨어하우스

  • 호환성
  • SaaS
