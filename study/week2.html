Introduction

클라우드 컴퓨팅이란?

인터넷 ‘너머’에 존재하는 클라우드 사업자의 컴퓨터에서 정보처리를 하는 서비스

“인터넷을 통한 정보처리 서비스”

클라우드 컴퓨팅의 장점

  • 경제성
  • 유연성 : 리소스를 필요할 때, 필요한 만큼만 확장/축소
    • 사용한 양에 대해서만 지불
    • 거대한 규모의 경제
  • 가용성: 장애 발생시 계속 사용 가능
  • 빠른 구축 속도
  • 손쉬운 글로벌 서비스
    • 명 분 만에 전 세계에 배포
  • 강력한 보안

클라우드 컴퓨팅의 단점

  • 생각보다 비싼 비용
  • 점점 커지는 클라우드 의존성
  • 데이터 보관의 불안함

클라우드 컴퓨팅 이용 모델 - 배포 모델

  • deployment model
  • private cloud
    • 독점적으로 사용되는 클라우드 컴퓨팅 리소스
    • hosted private cloud
      • 클라우드 사업자가 기업 사용자별로 클라우드 환경 제공
  • community cloud
    • 공동의 목적을 가진 기업/조직들이 클라우드 시스템을 형성하여 데이터 센터에서 공동 운영하는 형태
  • public cloud
    • 클라우드 사업자가 시스템 구축
  • hybrid cloud
    • private, public, community 서비스들과 on-premise 시스템을 연계시켜 활용하는 시스템
  • On-premise
    • 회사 내에 자체적으로 데이터 센터 보유
    • 시스템 구축, 운용까지 직접 수행

Serivces

클라우드 컴퓨팅의 용도

가상 서버(virtual server)

  • 로드 밸런싱(load balancing) - 이중화, 부하 분산
    • 하나의 리전 내의 복수 개의 zone에 서버를 설치하여 부하를 분산시킴
  • 오토 스케일링(auto scaling)
    • 가상 서버 갯수를 자동으로 증감시킴
  • 스냅 샵(snap shot)
    • 서버 디스크 백업

스토리지 서비스(storage)

  • 데이터 보관, 백업, 파일 서버, 재해 복구 등의 용도로 이용
  • S3 (simple storage service)
    • 여러 개의 데이터 센터에 데이터를 분산 저장
  • EBS( Elastic Block Store)
    • 엄격한 응답 시간이 요구되는 데이터베이스 용도
    • 블록 단위 액세스
  • EFS( Elastic File System)
    • 기업의 파일 서버
    • 파일 단위 액세스

네트워크 서비스 - VPN, 전용선

  • Amazon VPC(virtual private cloud)
    • AWS위에 가상 네트워크를 만들어 개인 클라우드처럼 사용할 수 있는 서비스
  • 전용선
    • on-premise의 네트워크와 AWS 네트워크를 연결하기 위한 서비스
  • 보안 기능
    • 필터링을 통해 불필요한 통신 차단
    • 보안 그룹 : 송수신 저액 설정 → 가상 서버에 대한 접속 제어
  • Amazon Route 53 : DNS 서비스

데이터 베이스

  • RDBMS
  • 데이터웨어하우스
  • NoSQL

Technologies

가상화 기술

  • 하이퍼바이저 형
  • 호스트 OS 형
  • 컨테이너 형
  • VLAN(virtual LAN)
    • 하나의 물리적 network → 여러 개의 논리적 network
    • 클라우드 서비스와 데이터 센터 사이에 VLAN 이용 → private 환경 구축
  • VPN( virtual private network)
    • 불특정 다수가 이용하는 네트워크에 가상으로 전용선과 같은 사설망을 연결하는 기술
    • IPsec protocol : on-premise system이 인터넷을 통해 VPN에 연결할 때 사용

분산처리 기술

: 대량의 빅데이터를 여러 서버에 분산하여 동시에 병렬로 빠르고 효율적으로 처리하는 기술

  • 클러스터링 : 여러 개의 서버를 결합하여 하나의 컴퓨터로 보이게 함

데이터베이스 기술

  • RDB
  • NoSQL
    • 대량의 데이터를 분산시켜 고속으로 처리하는 분산 데이터베이스

스토리지 기술

  • Block storage
    • 일정한 크기의 block으로 나눈 logical volume을 block 단위로 액세스
  • File storage
    • 파일 단위로 저장
  • Object storage
    • 데이터를 객체 단위로 처리
    • HTTP 프로토콜 기반의 REST(Representational State Transfer) 형식의 API 사용

API

: 프로그램이 가진 기능이나 리소스를 외부의 다른 프로그램이 호출하여 이용하기 위한 명령이나 함수, 데이터 형식 등을 정한 규약
